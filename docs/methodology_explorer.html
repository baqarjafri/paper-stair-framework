How the scores come together

STAIR scores a learning management system on its architectural readiness to host an autonomous teaching agent. The pipeline takes raw evidence from the Moodle source tree, scores 28 sub-dimensions on a 0–4 ordinal scale, consolidates those scores into a 5×5 Table 3 matrix, and averages per level. Every step is shown below.

STEP 1 · RUBRIC

Formal instrument defines 28 sub-dimensions grounded in Zimmerman SRL, Talebirad-Nadiri multi-agent, and Chu educational LLM theory.

→

STEP 2 · SCORE

Each sub-dimension scored 0–4 with evidence from Moodle source (file paths, class names, constants).

→

STEP 3 · CONSOLIDATE

Raw 0–4 scores mapped to percentages and averaged into a 5×5 (level × dimension) matrix.

→

STEP 4 · LEVEL AVG

Each level's five consolidated cells averaged into the headline score.

The architectural finding behind the L2→L3 cliff

Moodle scores well at L1 and L2 (the data layers exist) but collapses at L3 Agent Reasoning. The mechanism is not a missing feature — it is a missing connection. The AI subsystem and the analytics framework live in parallel, with zero shared code.
